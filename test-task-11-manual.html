🎨 Task 11: Visual Feedback & Animations - Manual Testing Guide

Purpose: This guide will help you manually test all visual feedback and animation features implemented in Task 11.

🚀 Launch Dashboard in New Tab

Test 1: Task Reordering Animation

Req 10.6 300ms Smooth Visual Animation

  1. Open the dashboard (click button above)
  2. Add at least 3 tasks (e.g., "Task 1", "Task 2", "Task 3")
  3. Click the ↑ button on the second task
  4. Observe the smooth animation as tasks swap positions
  5. Click the ↓ button on the first task
  6. Verify the animation is smooth and visible
✓ Expected: Tasks should smoothly transition to their new positions with a visible 300ms animation. The movement should feel natural and not jarring.
💡 Note: The animation uses CSS transitions with transform 0.3s ease for smooth movement.

Test 2: Theme Transition Speed

Req 10.1 ≤300ms Theme Toggle Transition

  1. Locate the theme toggle button (🌓) in the top-right corner
  2. Click the button to switch from light to dark mode
  3. Observe the transition speed - it should feel instant but smooth
  4. Click again to switch back to light mode
  5. Toggle multiple times to verify consistency
✓ Expected: The theme transition should complete within 300 milliseconds. Colors should smoothly fade from one theme to another without any jarring changes.
💡 Note: All elements have transition: background-color 0.3s ease, color 0.3s ease applied.

Test 3: Duplicate Task Notification

Req 10.2 ≥2 seconds Duplicate Prevention Notification

  1. Add a task with text "Buy milk"
  2. Try to add the same task again: "Buy milk"
  3. A red notification should appear at the top: "This task already exists!"
  4. Count how long the notification is visible (should be at least 2 seconds)
  5. Try with different case: "BUY MILK" - should also show notification
  6. Try editing a task to duplicate text - should also show notification
✓ Expected: A red notification banner appears at the top center of the screen, displays for exactly 2 seconds, then smoothly disappears. The notification should slide down with animation.
💡 Note: Duplicate detection is case-insensitive and trims whitespace. The notification uses a slideDown animation for smooth appearance.

Test 4: Settings Success Indicators

Req 10.3 2 seconds Save Success Feedback

  1. Scroll down to the Settings section
  2. Enter your name in the "Your Name" field (e.g., "Alice")
  3. Click the "Save" button
  4. Look for the green "✓ Saved" indicator next to the button
  5. Verify it displays for about 2 seconds then disappears
  6. Enter a timer duration (e.g., "30")
  7. Click "Save" and verify the success indicator appears
✓ Expected: A green "✓ Saved" indicator appears next to the Save button immediately after clicking. It remains visible for 2 seconds then smoothly disappears.
💡 Note: The greeting should update immediately with your name. The timer should reset to the new duration.

Test 5: Error State Highlighting

Req 10.4 3 seconds Invalid Input Highlighting

  1. In the Settings section, find the "Timer Duration" field
  2. Enter an invalid value: 0 (below minimum)
  3. Click "Save"
  4. Observe the input field - it should have a red border and light red background
  5. An error message should appear below: "Duration must be between 1 and 120 minutes"
  6. Try other invalid values: 150 (above maximum), abc (not a number)
  7. Verify error state persists for about 3 seconds
✓ Expected: The input field is highlighted with a red border (#f44336) and light red background. An error message appears below the field. Both disappear after 3 seconds.
💡 Note: Valid range is 1-120 minutes. The error class is applied to the input element and removed after 3 seconds.

Test 6: Hover States for All Controls

Req 10.5 Interactive Control Feedback

  1. Theme Toggle: Hover over the 🌓 button - should scale up and show enhanced shadow
  2. Move Buttons: Hover over ↑ and ↓ buttons on tasks - background should change
  3. Settings Buttons: Hover over Save buttons - should darken to a deeper green
  4. Input Fields: Click into any input field - border should change color (green for tasks/name, blue for links)
  5. Task Buttons: Hover over Complete, Edit, Delete buttons - background should change
  6. Timer Buttons: Hover over Start, Stop, Reset - colors should darken
✓ Expected: All interactive controls provide visual feedback on hover. Buttons should change color or scale, input fields should show focus borders.
💡 Note: Hover states use CSS transitions for smooth color changes. Active states (when clicking) also provide scale feedback.
